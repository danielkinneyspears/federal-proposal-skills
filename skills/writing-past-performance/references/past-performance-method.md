# Past Performance Method

How federal evaluators assess past performance, and how to select and write
references that score well.

## The three tests

Federal past performance evaluation asks three questions: Is the cited work
recent enough to be meaningful? Is it relevant to what is being bought now? Was
it performed well? A reference is only strong if it passes all three. Capture
teams over-index on quality ("we got an Exceptional rating") and under-index on
relevancy — but an Exceptional rating on work unlike the current requirement
carries little weight.

## Recency

Recency is whether the work is current enough to predict future performance.
Section L usually sets the window — work performed or active within a stated
number of years (commonly three to five) from the solicitation date. Work
outside the window may be excluded entirely. Confirm the window before selecting
anything.

## Relevancy

Relevancy is how closely the prior work resembles the current requirement.
Section M defines the dimensions; common ones:

- **Scope** — the same kind of work and disciplines.
- **Magnitude / size** — comparable dollar value and level of effort.
- **Complexity** — comparable technical, managerial, or operational difficulty.
- **Contract type** — the same or similar (FFP, cost-reimbursement, IDIQ).
- **Customer / environment** — the same or a similar agency, mission, or
  operating environment.
- **Place of performance** — comparable locations or conditions.

Section M often defines relevancy *ratings* (for example: Relevant, Somewhat
Relevant, Not Relevant) keyed to how many dimensions match. Read that scale and
select references that reach the top of it.

## Quality

Quality is how well the prior work was performed. Evidence, strongest first:

- **CPARS ratings** — the government's official performance record. Often the
  primary evidence; evaluators may pull CPARS directly.
- **Past Performance Questionnaires (PPQs)** — reference forms the prior customer
  completes, used where CPARS is unavailable or to supplement it.
- **Award-fee or incentive history** — fees earned signal rated performance.
- **Quantified outcomes** — measurable results delivered (schedule met, cost
  saved, performance targets exceeded).
- **Commendations and awards** — letters, recognition.

## Selecting references

1. Score every library contract on recency, relevancy, and quality.
2. Within the allowed count, select the set that is strongest across all three —
   prioritizing relevancy, because it is most often the weak link and the
   discriminator.
3. Seek breadth: a set that collectively covers the major dimensions of the
   current requirement is stronger than three references that all demonstrate
   the same one thing.
4. Prefer references where the offeror was the prime, unless the solicitation
   values subcontractor experience equally.
5. A famous, large contract that is not relevant is not a good reference. Choose
   relevance.

## The relevance mapping

The relevance mapping is what turns a project description into a scoring
argument. For each reference, state — dimension by dimension — how the prior
contract resembles the current requirement:

> *This reference's scope (enterprise network operations for a 12,000-user DoD
> component) directly matches the current requirement's scope; its value
> ($48M over five years) is comparable in magnitude; its complexity (24/7
> operations across three security domains) matches; and it was performed for
> the same customer environment.*

Do not leave the evaluator to infer the resemblance. Map it explicitly, on the
dimensions Section M names, so the evaluator can assign the top relevancy rating
without doing the analysis themselves.

## Substantiating quality

Replace every quality assertion with evidence. Not "we consistently exceeded
expectations" but "the offeror earned Exceptional CPARS ratings in all five
evaluation periods and 96% of available award fee." Cite the source so the
evaluator can verify it.

## Handling performance problems

If a candidate reference had a real performance problem:

- Evaluators frequently have independent CPARS access. An omitted problem the
  evaluator finds reads as concealment and damages credibility across the whole
  volume.
- If the reference is used, address the problem directly: what happened, the
  root cause, the corrective action taken, and the result — ideally showing the
  rating recovered.
- If the problem is severe and a stronger reference exists, select the stronger
  reference instead.
- Never misrepresent a rating or outcome.

## Teaming partner and subcontractor references

When teaming partners contribute past performance:

- Confirm the solicitation allows partner/subcontractor references and how it
  weighs them relative to prime references.
- Make clear which entity performed the cited work and in what role.
- Tie the partner's reference to the scope that partner will perform on this
  effort — a partner's past performance is most persuasive for the part of the
  work that partner will actually do.
